ПРИЛОЖЕНИЕ №3: ДНЕВНИК ДАВЛЕНИЯ
========================================================================
------------------------------------------------------------------------

1. НАЗНАЧЕНИЕ ПРИЛОЖЕНИЯ
------------------------------------------------------------------------
Дневник давления — это медицинское приложение для записи артериального давления, пульса
и самочувствия. Оно помогает пользователю регулярно сохранять измерения, смотреть
историю, отслеживать динамику на графике и получать напоминания о необходимости измерить
давление.

2. ОСНОВНЫЕ ВОЗМОЖНОСТИ
------------------------------------------------------------------------
• добавление верхнего и нижнего давления;
• добавление пульса и заметки;
• проверка корректности введённых данных;
• история измерений;
• удаление отдельных записей;
• очистка всей истории;
• график верхнего и нижнего давления;
• расчёт средних значений;
• несколько напоминаний в день;
• выбор дней недели для каждого напоминания;
• редактирование и удаление напоминаний.

3. ИНТЕРФЕЙС
------------------------------------------------------------------------
Интерфейс выполнен в спокойных тёмных цветах. Это подходит для медицинского приложения,
потому что экран не выглядит перегруженным и не раздражает глаза. Основные блоки
оформлены как карточки: ввод измерения, результат, статистика и график.

На главном экране пользователь видит поля для давления, пульса и заметки. Ниже
отображается результат последнего измерения, общая статистика и график. Отдельные кнопки
открывают историю и настройки напоминаний.

4. ЗАПИСЬ ДАВЛЕНИЯ
------------------------------------------------------------------------
Пользователь вводит верхнее давление, нижнее давление, пульс и при желании заметку.
Перед сохранением приложение проверяет данные: значения должны быть числовыми,
находиться в допустимом диапазоне, а нижнее давление должно быть меньше верхнего.

После сохранения запись добавляется в начало истории. На экране обновляется статистика и
график. Также выводится краткая интерпретация результата: нормальное давление,
повышенное давление или высокое давление.

5. ГРАФИК И СТАТИСТИКА
------------------------------------------------------------------------
График реализован через собственный View. Он рисует две линии: верхнее давление и нижнее
давление. Для удобства отображаются последние записи, чтобы пользователь видел динамику
показателей.

Статистика показывает последнее измерение, среднее давление, средний пульс, количество
записей, а также максимальное и минимальное верхнее давление.

6. НАПОМИНАНИЯ
------------------------------------------------------------------------
Напоминания можно настраивать полностью. Пользователь может создать несколько
напоминаний на разные времена суток, например утром, днём и вечером. Для каждого
напоминания отдельно выбираются дни недели.

Напоминания сохраняются списком. Каждое напоминание можно изменить или удалить. Когда
наступает нужное время, приложение показывает push-уведомление с предложением измерить
давление.

7. РАБОТА ВНУТРИ ПРИЛОЖЕНИЯ
------------------------------------------------------------------------
Для хранения измерений и напоминаний используется SharedPreferences. Для уведомлений
используется NotificationManager, а для запуска в нужное время — AlarmManager. После
срабатывания напоминание планируется заново на следующий подходящий день.

Также приложение создаёт NotificationChannel для современных версий Android и
запрашивает разрешение POST_NOTIFICATIONS на Android 13 и выше.

8. ИТОГ
------------------------------------------------------------------------
Приложение показывает медицинский сценарий: ввод данных, хранение истории, визуализация,
расчёт статистики и работа с фоновыми напоминаниями. Такой проект хорошо подходит для
лабораторной работы по разработке медицинского мобильного приложения.
